Lab 1 — Reflected XSS into HTML context with nothing encoded

🟡 Medium 📅 2026-05-20

📋 Finding Summary

FieldValue
TypeWeb App
Severity🟡 Medium
CVE / CWECWE-79
Date2026-05-20

🔍 Description

Reflected Cross-Site Scripting (XSS) occurs when user-supplied input is echoed back into the HTML response without sanitization or encoding. In this lab, the blog search functionality reflects the search query directly into the page HTML. Because the application performs no output encoding, an attacker can inject arbitrary JavaScript that executes in the victim’s browser in the context of the vulnerable site.

🎯 End Goals

  • Inject a script into the search field that calls the alert function
  • Confirm execution by observing the browser alert dialog
  • Solve the lab

🪜 Steps to Reproduce

  1. Access the lab and navigate to the blog search functionality
  2. Enter an XSS payload into the search field and submit
  3. Observe that the injected script executes in the browser
  4. Confirm the lab is marked as solved

🧠 Analysis

Step 1 — Access the lab
The lab environment loaded a blog application with a search field. The lab status showed “Not solved”, indicating no prior interaction.

Step 2 — Inject the XSS payload
The payload <Script>alert("test")</Script> was entered into the search box and submitted. The application reflected the input directly back into the HTML response without any encoding or sanitization, allowing the script tag to be interpreted by the browser.

Step 3 — alert() fires
Upon submission, the browser executed the injected script and displayed an alert dialog with the text “test”. This confirms that the search parameter is vulnerable to reflected XSS — unsanitized user input is rendered as executable JavaScript in the page.

Step 4 — Lab solved
After dismissing the alert, the page displayed “Congratulations, you solved the lab!” and the lab status updated to Solved, confirming successful exploitation.

🎯 Impact

An attacker who can trick a victim into clicking a crafted link containing a malicious payload could execute arbitrary JavaScript in the victim’s browser. This can be used to steal session cookies, capture credentials, perform actions on behalf of the user, redirect to phishing pages, or deliver malware — all within the context of the trusted domain.
