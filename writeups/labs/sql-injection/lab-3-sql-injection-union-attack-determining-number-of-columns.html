Lab 3 — SQL Injection UNION Attack — Determining Number of Columns

🟡 Medium 📅 2026-05-10

FieldValue
TypeWeb App

🔍 Description

The /filter endpoint accepts a category parameter that is passed directly into a SQL query without sanitization. By injecting a UNION SELECT statement, an attacker can append a second query to the original. However, for a UNION attack to succeed, the injected query must return the same number of columns as the original query.

The technique used here is to incrementally add NULL values to the UNION SELECT clause until the application returns a valid response instead of an error, revealing the exact number of columns in the original query. Three columns were confirmed.

🎯 End Goals

  • Determine the exact number of columns returned by the original query
  • Use incremental UNION SELECT NULL injection until a valid response is received
  • Confirm 3 columns and solve the lab

🪜 Steps to Reproduce

  1. Navigate to the web shop and browse to any product category.
  2. Observe the URL: /filter?category=Gifts — the category is passed as a GET parameter.
  3. Open Burp Suite and enable proxy intercept.
  4. Capture the GET /filter?category=Gifts request.
  5. Inject ' UNION SELECT NULL-- into the category parameter — the application returns an error, indicating 1 column is incorrect.
  6. Increment to ' UNION SELECT NULL,NULL-- — still returns an error, 2 columns is incorrect.
  7. Increment to ' UNION SELECT NULL,NULL,NULL-- — the application returns a valid response, confirming 3 columns.
  8. Lab solved.

🧠 Analysis

Step 1 — Inject 1 NULL — error response

Sent ' UNION SELECT NULL-- in the category parameter. The application returned an internal server error, confirming the original query does not return 1 column.

Step 2 — Inject 2 NULLs — error response

Incremented to ' UNION SELECT NULL,NULL--. The application returned another error, confirming 2 columns is also incorrect.

Step 3 — Inject 3 NULLs — valid response

Incremented to ' UNION SELECT NULL,NULL,NULL--. The application returned a valid 200 response with product results, confirming the original query returns 3 columns.

Step 4 — Lab solved

Browser displayed the lab solved confirmation banner.

🎯 Impact

Determining the number of columns is the first step in a UNION-based SQL injection attack. Once confirmed, an attacker can proceed to extract data from other database tables by injecting meaningful values in place of the NULL placeholders, potentially exposing sensitive information such as usernames, passwords, and application data.
