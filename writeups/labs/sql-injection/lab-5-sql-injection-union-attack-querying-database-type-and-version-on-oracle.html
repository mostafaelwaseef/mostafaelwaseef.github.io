Lab 5 — SQL Injection UNION Attack — Querying Database Type and Version on Oracle

🟡 Medium 📅 2026-05-10

📋 Finding Summary

FieldValue
TypeWeb App
Severity🟡 Medium
CVE / CWECWE-89 — SQL Injection
Date2026-05-10

🔍 Description

The application's product filter functionality is vulnerable to SQL injection. The category parameter is passed unsanitized into a backend Oracle SQL query. By injecting a UNION-based payload, an attacker can append arbitrary SELECT statements to the original query and retrieve data from other tables or built-in Oracle views.

On Oracle databases, every SELECT statement must include a FROM clause. The dual table is a built-in single-row Oracle table used to satisfy this requirement when no real table is needed. Additionally, Oracle exposes the v$version view which contains database version and banner information.

This lab required three stages: first determining the number of columns returned by the original query, then confirming which columns accept string data, and finally querying the Oracle-specific v$version view to retrieve the database version.

🎯 End Goals

  • Determine the number of columns returned by the original query
  • Confirm which columns accept string data
  • Query the Oracle v$version view to retrieve the database version string
  • Solve the lab by displaying the version in the response

🪜 Steps to Reproduce

  1. Navigate to the shop and select any product category to observe the category parameter in the URL.
  2. Send the request to Burp Repeater.
  3. Inject ' UNION select NULL from dual-- to test for a single column — the server returns an Internal Server Error, indicating more than one column is returned.
  4. Inject ' UNION select NULL,NULL from dual-- — the page renders normally, confirming 2 columns.
  5. Inject ' UNION select 'a',NULL from dual-- — a string value appears in the response, confirming column 1 accepts strings.
  6. Inject ' UNION select 'a','a' from dual-- — both columns render string values, confirming both columns accept strings.
  7. Inject ' UNION select banner,NULL from v$version-- — the Oracle database version is returned in the response.
  8. The lab is marked as solved once the version string appears.

🧠 Analysis

Step 1 — 1-column UNION attempt — Internal Server Error

Sent ' UNION select NULL from dual-- in Burp Suite Repeater. The server returned an Internal Server Error, confirming the original query returns more than 1 column.

Step 2 — 2-column UNION succeeds — page renders

Incremented to ' UNION select NULL,NULL from dual--. The page rendered normally with products, confirming the original query returns 2 columns.

Step 3 — Column 1 confirmed as string type

Sent ' UNION select 'a',NULL from dual--. The value a appeared in the product listing, confirming column 1 accepts string data.

Step 4 — Both columns confirmed as string type

Sent ' UNION select 'a','a' from dual--. Both injected string values appeared in the response, confirming both columns accept strings.

Step 5 — Final payload in Burp Repeater — Oracle version retrieved

Sent ' UNION select banner,NULL from v$version--. The Oracle version string Oracle Database 11g Express Edition Release 11.2.0.2.0 - 64bit Production was returned in the response.

Step 6 — Lab solved banner

Browser displayed the lab solved confirmation banner.

🎯 Impact

An attacker can leverage this vulnerability to extract sensitive information from the Oracle database. By querying the v$version view, the exact database version is revealed, enabling targeted exploitation of version-specific vulnerabilities. Further UNION-based queries could expose table names, user credentials, or application data from any table accessible to the database user running the query.
