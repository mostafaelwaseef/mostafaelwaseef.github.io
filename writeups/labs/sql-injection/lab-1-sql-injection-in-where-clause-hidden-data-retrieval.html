Lab 1 — SQL Injection in WHERE Clause — Hidden Data Retrieval

🟡 Medium 📅 2026-05-10

📋 Finding Summary

FieldValue
TypeWeb App
Severity🟡 Medium
CVE / CWECWE-89 — SQL Injection
Date2026-05-10

🔍 Description

The /filter endpoint accepts a category parameter which is passed directly into a SQL WHERE clause without sanitization. The application filters products by category with an additional AND released = 1 condition to hide unreleased items. By injecting a single quote (') to break out of the SQL string and -- to comment out the remainder of the query, the released = 1 filter is bypassed — causing the database to return all products including hidden ones.

The underlying SQL query is likely:

SELECT * FROM products WHERE category = '[input]' AND released = 1

After injection, it becomes:

SELECT * FROM products WHERE category = 'Gifts'-- AND released = 1

🎯 End Goals

  • Inject into the category parameter to bypass the released = 1 filter
  • Retrieve hidden (unreleased) products from the database
  • Confirm successful injection by observing the hidden item appear in the response

🪜 Steps to Reproduce

  1. Navigate to the web shop and browse to the Gifts category.
  2. Observe the URL: /filter?category=Gifts — the category is passed as a GET parameter.
  3. Open Burp Suite and enable proxy intercept.
  4. Refresh the page and capture the GET request.
  5. Note the normal response shows 3 products (released items only).
  6. Attempted &released=0 parameter manipulation — no effect.
  7. Modified the category parameter to: category=Gifts' --
  8. Forwarded the modified request.
  9. Page now shows 4 products, including the hidden item "High-End Gift Wrapping".
  10. Page title reflects the injected value ("Gifts' --"), confirming successful injection.

🧠 Analysis

Step 1 — Original request intercepted in Burp Suite

Captured the GET /filter?category=Gifts request in Burp Suite proxy intercept before any modification.

Step 2 — Normal page before injection (3 products)

Browsed the Gifts category without any injection. The page returned 3 products — only the released items visible to normal users.

Step 3 — Injected request with Gifts' -- payload

Modified the category parameter to Gifts' -- in Burp Suite and forwarded the request. The single quote breaks out of the SQL string and -- comments out the AND released = 1 condition.

Step 4 — Hidden product revealed (4 products)

The page now returned 4 products, including the hidden item "High-End Gift Wrapping". The page title reflected the injected value, confirming successful injection.

🎯 Impact

An attacker can bypass application-level access controls on data visibility. In this case, unreleased products are exposed to unauthenticated users. In a real-world scenario, this class of vulnerability can be escalated to extract sensitive data from other tables, bypass authentication, or perform full database enumeration via UNION-based or blind SQL injection.
