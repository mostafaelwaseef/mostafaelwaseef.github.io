Lab 8 — SQL Injection Attack — Listing Database Contents on Oracle

🔍 Description

This lab demonstrates a UNION-based SQL injection vulnerability in a product category filter on an Oracle database. Unlike standard databases, Oracle requires FROM DUAL in every SELECT statement and uses ALL_TABLES and ALL_TAB_COLUMNS instead of information_schema for schema enumeration. An attacker can exploit this to enumerate all tables, retrieve column names, and extract credential data from the application database.

🎯 End Goals

  • Determine the number of columns returned by the query and confirm which columns accept string data
  • Confirm the Oracle database version using v$version
  • Enumerate table names using Oracle's ALL_TABLES view to identify a users table
  • Retrieve column names from that table using ALL_TAB_COLUMNS
  • Dump usernames and passwords and log in as the administrator user

🪜 Steps to Reproduce

  1. Intercept the product category filter request in Burp Suite
  2. Probe column count with increasing NULLs; add FROM DUAL when two NULLs still error (Oracle syntax requirement)
  3. Test each column individually for string compatibility using 'a' and NULL
  4. Retrieve the database version using v$version
  5. Query ALL_TABLES to list all table names and identify the users table
  6. Query ALL_TAB_COLUMNS filtered on the users table to retrieve column names
  7. Dump credentials with a UNION SELECT targeting the identified table and columns
  8. Log in to the application as administrator

🧠 Analysis

Step 1 — Column count confirmation

The lab home page confirmed the target was unsolved. The product category filter parameter was intercepted in Burp Suite. An initial probe with ' UNION SELECT NULL-- returned a 500 error, indicating the query returns more than one column. A follow-up with two NULLs (' UNION SELECT NULL,NULL--) also returned a 500 — the query was syntactically valid in column count but Oracle requires a FROM clause in every SELECT statement, which standard syntax omits. Adding FROM DUAL (' UNION SELECT NULL,NULL FROM DUAL--) returned a 200 OK, confirming exactly two columns.

' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL FROM DUAL--

Step 2 — String type confirmation

With two columns established, each was tested individually for string compatibility by injecting a string literal into one column while keeping the other as NULL. The payload ' UNION SELECT 'a',NULL FROM DUAL-- confirmed the first column accepts strings; ' UNION SELECT NULL,'a' FROM DUAL-- confirmed the second. Both columns could be used to extract text data.

' UNION SELECT 'a',NULL FROM DUAL--
' UNION SELECT NULL,'a' FROM DUAL--

Step 3 — Database version confirmation

With string-compatible columns confirmed, the Oracle database version was retrieved using the v$version view, which contains the BANNER column with version information. The response confirmed the target was running Oracle Database 11g Express Edition Release 11.2.0.2.

' UNION SELECT BANNER,NULL FROM v$version--

Step 4 — Table enumeration via ALL_TABLES

Before querying, a Google search was performed to review the ALL_TABLES view schema and identify which columns were available and relevant to use. The payload was constructed and executed, returning the full list of accessible table names. Reviewing the output revealed a table named USERS_YEXJB.

' UNION SELECT TABLE_NAME,NULL FROM ALL_TABLES--

Step 5 — Column enumeration via ALL_TAB_COLUMNS

A Google search was performed to review the ALL_TAB_COLUMNS view schema and identify which columns were available and relevant to use. The view was queried filtered on USERS_YEXJB. The response revealed two credential columns: USERNAME_FDWBVY and PASSWORD_CREDSF.

' UNION SELECT COLUMN_NAME,NULL FROM ALL_TAB_COLUMNS WHERE TABLE_NAME='USERS_YEXJB'--

Step 5 — Dump credentials and login

A final UNION SELECT retrieved both columns simultaneously from USERS_YEXJB. The response returned all user credentials including administrator:bqljce893yl80a1vmx5r. The administrator account was used to log in, solving the lab.

' UNION SELECT USERNAME_FDWBVY,PASSWORD_CREDSF FROM USERS_YEXJB--

🎯 Impact

An attacker with access to this SQL injection point can fully enumerate the Oracle database schema — listing all tables and columns — and extract any stored data including plaintext credentials. This enables complete account takeover, including administrative access, and potentially lateral movement if credentials are reused elsewhere.
