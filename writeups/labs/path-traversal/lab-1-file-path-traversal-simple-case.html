Lab 1 — File path traversal, simple case

🔴 High 📅 2026-07-12

📋 Finding Summary

FieldValue
TypeWeb App
Severity🔴 High
CVE / CWECWE-22: Improper Limitation of a Pathname to a Restricted Directory
Date2026-07-12

🔍 Description

Path traversal (also known as directory traversal) allows an attacker to read arbitrary files on the server by manipulating file path parameters. In this lab, the application uses a filename parameter in the /image endpoint to load product images. No sanitization or restriction is applied to this parameter, allowing an attacker to supply traversal sequences (../) to walk up the directory tree and access sensitive files outside the web root.

🎯 End Goals

  • Exploit the path traversal vulnerability in the product image endpoint
  • Retrieve the contents of /etc/passwd from the server

🪜 Steps to Reproduce

  1. Browse to any product page on the shop
  2. Open browser DevTools and inspect the product image element to identify the vulnerable filename parameter
  3. Modify the filename parameter value to a path traversal payload pointing to /etc/passwd
  4. Observe the server returns the file contents, solving the lab

🧠 Analysis

Step 1 — Browse the shop and identify the target product

Opened the lab environment and browsed to the product page for “Babbage Web Spray” at /product?productId=1. The page loaded normally with product images displayed.

Step 2 — Identify the vulnerable filename parameter via DevTools

Opened browser DevTools and used the Inspector to examine the HTML source of the product page. The product image was found to load via the endpoint /image?filename=22.jpg, directly exposing a filename parameter with no apparent sanitization.

Step 3 — Inject traversal payload and solve the lab

Modified the src attribute of the image element in DevTools to /image?filename=../../../etc/passwd. The server processed the traversal sequence without restriction, walking three directory levels up from the images directory to the filesystem root and returning the contents of /etc/passwd. The lab banner confirmed success with “Congratulations, you solved the lab!”

🎯 Impact

An attacker can read arbitrary files from the server filesystem, including sensitive system files (/etc/passwd, /etc/shadow), application configuration files containing credentials, private keys, and source code. On Windows systems, equivalent payloads using ..\ can expose files such as C:\Windows\win.ini. This can lead to full server compromise if credentials or keys are exposed.
