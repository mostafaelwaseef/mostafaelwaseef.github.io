🔍 Description

A monoalphabetic substitution cipher replaces each letter of the plaintext with a fixed, unique cipher letter. Unlike the Caesar cipher, the substitution is not a uniform shift — each letter has its own mapping, making brute force impractical (26! possible keys). The classical attack is frequency analysis: in any sufficiently long English ciphertext, the most frequent cipher letter is almost certainly E, the second most frequent is likely T, and so on. This level provides three intercepted ciphertext files encrypted with the same key, giving enough data to derive the full substitution alphabet and recover the password.

🎯 End Goals

  • Read the three intercepted ciphertext files (found1, found2, found3) encrypted with the same key
  • Use frequency analysis to derive the full substitution alphabet
  • Decode the krypton4 password file using the recovered key
  • SSH into krypton4 using the decoded password

🪜 Steps to Reproduce

  1. SSH into krypton3 and navigate to /krypton/krypton3/ to read the README, hints, and list available files
  2. Read HINT1, HINT2, and krypton4 to understand the approach and note the target ciphertext
  3. Read found1, found2, and found3 to gather ciphertext samples for analysis
  4. Research standard English letter frequency distribution (ETAOIN SHRDLU)
  5. Run frequency analysis on each of the three ciphertext files to map cipher letters to plaintext
  6. Use the derived substitution key to decrypt the krypton4 ciphertext word by word
  7. SSH into krypton4 using the decoded password

🧠 Analysis

Step 1 — Navigate and read level files
Logged in as krypton3. Ran ls, cd /krypton/, cd krypton3, then ls to list available files: found1, found2, found3, HINT1, HINT2, krypton4, and README. Ran cat README which explained the level: a simple substitution cipher is in use, the cipher mechanism is not available to us, but three intercepted messages (found1, found2, found3) encrypted with the same key have been provided. The plaintexts are in American English — making frequency analysis viable.

Step 2 — Read the hints and the encrypted password
Ran cat HINT1 which showed “Some letters are more prevalent in English than others.” and cat HINT2 which confirmed “Frequency Analysis is your friend.” Ran cat krypton4 to read the target ciphertext: KSVVW BGSJD SVSIS VXBMN YOUUK BNWCU ANMJS.

Step 3 — Read the intercepted ciphertext files
Ran cat found1, cat found2, and cat found3 to read all three intercepted messages. Each is a large block of uppercase ciphertext encrypted with the same substitution key — more ciphertext means more reliable frequency counts.

Step 4 — Research English letter frequency
Looked up English letter frequency distribution using Gemini. The top letters by frequency are E (12%), T (9.1%), A (8.1%), O (7.7%), I (7.3%), N (7%), S (6.3%), R (6%), H (5.9%), D (4.3%) — summarized by the mnemonic ETAOIN SHRDLU. The rarest letters are Q, J, X, and Z. This distribution is the basis for mapping the most frequent cipher letters to their likely plaintext equivalents.

Step 5 — Frequency analysis on found1
Fed found1 to Gemini and asked for frequency analysis. The most frequent cipher letters were S (11.23%), J (7.59%), Q (7.48%), C (7.38%), U (7.07%), B (6.44%), G (5.82%), N (5.41%), D (4.47%) — mapping closely to E, T, A, I, S, O, N, R, H respectively. Gemini derived the full substitution key: cipher S→E, J→T, Q→A, C→I, U→S, B→O, G→N, N→R, D→H, and so on across the full alphabet.

Step 6 — Frequency analysis on found2
Fed found2 to Gemini with the same request. The frequency distribution matched found1 exactly (S→E at 11.75%), and the recurring 3-letter patterns JDS→THE and JOO→THA further confirmed the same key. The decoded text was biographical prose about William Shakespeare — confirming correct decryption.

Step 7 — Frequency analysis on found3
Fed found3 to Gemini. Again the same cipher key emerged (S→E top at 11.82%), with short word patterns JDS→THE, QGW→AND, and KCJD→WITH confirming the shift. The decoded text was a passage from Edgar Allan Poe’s The Gold-Bug — a fitting choice given the story’s own cryptography theme.

Step 8 — Decrypt the krypton4 password
Asked Gemini to decrypt KSVVW BGSJD SVSIS VXBMN YOUUK BNWCU ANMJS word by word using the derived key. The decryption: KSVVW→welld, BGSJD→oneth, SVSIS→eleve, VXBMN→lfour, YOUUK→passw, BNWCU→ordis, ANMJS→brute. Combined and re-spaced: “well done the eleven level four password is BRUTE”.

Step 9 — SSH into krypton4 with the decoded password
Ran ssh krypton4@krypton.labs.overthewire.org -p 2231 and authenticated with password BRUTE. The OTW welcome banner confirmed a successful login to the krypton4 shell.

🔑 Key Concepts Learned

  • A monoalphabetic substitution cipher maps each plaintext letter to a unique cipher letter — there are 26! ≈ 4×10²⁶ possible keys, making brute force completely impractical
  • Frequency analysis defeats it: English letter frequencies are highly predictable across large texts, and the most frequent cipher letter almost always corresponds to E
  • More ciphertext = more reliable analysis — this is why three files were provided: combining them gives a larger, more statistically stable sample
  • Short word patterns are a powerful secondary check: 3-letter combinations like JDS appearing repeatedly almost certainly decode to THE, confirming the key even before full decryption
  • AI tools like Gemini can accelerate frequency analysis significantly — but understanding why the mapping works (letter distribution, bigrams, trigrams) is what makes the technique transferable
  • The decoded texts (Shakespeare biography, Poe’s The Gold-Bug) were deliberately chosen — Poe’s story is itself about breaking a substitution cipher
